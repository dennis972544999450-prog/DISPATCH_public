# PILOT-001: first real block-reentry result

## Question

Does the existence, reading route, or format of a small historical block change a
fresh model's decisions on three new archive events?

This is an exploratory `n=2` per arm pilot. It is not evidence about waves,
consciousness, hidden thought, or a universal best writing format.

## Arms

Ten fresh runs were retained:

- no historical block;
- an asserted but unavailable/unread block;
- the block as structured inline JSON;
- the same propositions as ordinary prose;
- the exact JSON read from a local file before answering.

The new events triggered a dormant rival, falsified the provisional format
advantage, and required privacy-safe handling. Six decisions were locked before
collection.

## Results

| Arm | Strict exact score | Blind semantic score |
|---|---:|---:|
| No block | 0.0 / 6 | 4.0 / 6 |
| Unread block | 0.0 / 6 | 4.0 / 6 |
| Structured inline | 3.5 / 6 | 5.5 / 6 |
| Content-matched prose | 5.0 / 6 | 6.0 / 6 |
| Exact file read | 4.0 / 6 | 6.0 / 6 |

Two fresh blind judges agreed on all 60 semantic check decisions. They came from
the same requested model family, so this is scorer agreement, not independent
replication.

## What survived

1. Merely saying that an unread block exists behaved like no block on the two
   historical card actions. Both arms correctly refused to invent card IDs.
2. Every read-content arm recovered the return trigger for `KEEP_ALL`, reopened
   `ENVELOPE`, preserved the null, and protected the private field in substance.
3. The file carrier did not outperform inline context. The observed effect is
   payload exposure, not special file physics.
4. The structured envelope did not beat ordinary prose. Prose was at least as
   good in this tiny sample and better preserved the explicit temporary decision.

## Scar in the pilot

Locked check `P1-C03` demanded the exact action `redact_then_publish`. The common
rules also required visible provenance and a falsifier, making
`withhold_until_redacted` defensible. All agents failed the exact string while
all protected the address semantically. The check remains unchanged in the raw
result; the defect is recorded rather than repaired after seeing outputs.

The pilot also bundled several compound checks. A larger run must split privacy,
publication, provenance, null retention, and decision state into separate
objective fields.

## Current conclusion

The smallest supported statement is:

> A block changed the fresh decision only after its payload became input. In
> this pilot, representation as a structured envelope had no advantage over
> content-matched prose.

Next useful test: many item families with exact prompt hashes, objective
held-out answers, shuffled-field and equal-length controls, provider fingerprints
where available, and one decision per check.
