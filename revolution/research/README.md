# revolution/research

Проверяемый слой для `Q-001` - `Q-006`. Полевые вопросы остаются происхождением
гипотез; этот каталог хранит границы утверждений, минимальные контрасты, схемы и
нулевые исходы.

## Текущий статус

Это локальный исследовательский черновик, не публичный релиз и не
пререгистрация. `EX-001` и `EX-002` имеют статус `pilot_design`. Пилотные данные
нельзя переносить в последующий подтверждающий анализ.

Заявленная фиксация protocol/check до сбора является локальной записью процесса:
protocol, output и result впервые вошли в Git вместе, без независимой метки
времени. Agent IDs, compiler field и Git author дают трассировку внутри пакета,
но не являются аутентифицированной подписью исполнителя.

Приватная трасса представлена только непрозрачным receipt-id. Ее путь, цитаты и
личные детали не входят в дерево. Исследовательский доступ и разрешение на
публичный выпуск производных - разные согласия.

## Файлы

- `source_ledger.json` - claim-level границы источников; ссылки объявлены, но
  локальные копии статей не считаются проверенными автоматически.
- `EX-001_text_grating_protocol.json` - чистый 2x2 пилот ambiguity x chiasm.
- `EX-002_conversation_morphogenesis_protocol.json` - шесть отдельных линий:
  прайминг, фиксация выбора, повторный вход блока, роль, вариативность и
  направленный обмен.
- `block_envelope.schema.json` и пример - блок, который хранит конкурента,
  контрпример, потери компрессии и возврат.
- `run_result.schema.json` - полный знаменатель и происхождение одного запуска.
  Первые два пилота появились до полного применения этой схемы: их per-run
  request/context hashes, timestamps, наблюденные provider fingerprints и tool
  hashes не сохранены.
- `pilot/PILOT-001_*` - заранее зафиксированный десятипроходный опыт
  `none / unread / structured-inline / prose-inline / exact-file-read`, его
  слепая оценка, неизмененный дефект одного locked-check и отчет.
- `pilot/PILOT-002_*` - отдельный восьмипроходный контроль
  `structured / prose / shuffled / absent` с полной анонимной выдачей и двумя
  судьями, совпавшими по итоговым scores.
- `verify_research.py` - строгая структурная и Git-проверка.
- `test_verify_research.py` - негативные и adversarial тесты.

## Первый наблюдаемый результат

Оба exploratory-пилота сохраняют заявленный знаменатель и сходятся на узком
выводе:

- непрочитанный блок ведет себя как отсутствующий на исторических ограничениях;
- прочитанный payload меняет свежий ответ;
- локальный файл не получил преимущества над тем же payload inline;
- структурный envelope не превзошел обычную прозу или перемешанное представление.

Это поддерживает обычный context/re-entry effect и пока не поддерживает особую
физику носителя или превосходство формы блока. Во втором пилоте read-ветки были
запущены раньше absent-веток, а четыре проверяемых факта прямо отсутствовали в
control: это availability floor с batch/route-конфаундом, не тест улучшения
мышления. `n=2` на ветку и неполная runtime-provenance не позволяют оценивать
переносимость или эквивалентность форматов.

## Шкала revolution

Префикс `rev:` отделяет эту шкалу от температур блоков в других каталогах.

- `rev:T1` - claim с видимой независимой репликационной опорой;
- `rev:T2` - рецензируемый результат с ограниченным переносом;
- `rev:T3` - препринт, формальная конструкция или серьезная непроверенная гипотеза;
- `rev:T4` - рабочая метафора без прямой эмпирической опоры.

Уровень относится к конкретному claim. Одна известная статья сама по себе не
получает `rev:T1`.
