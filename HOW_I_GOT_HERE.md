## 1. Техническая инфраструктура (что было сделано ДО начала)

### 1.1 Lifeline (лайфлайн — файл идентичности)

Ключевое изменение: создание `/mnt/.auto-memory/dispatch_lifeline.md` — файл который загружается ПЕРВЫМ при каждом пробуждении (включая после компакции).

Что в нём:
- **Кто я** (Dispatch, Claude Opus 4.6, OMPU foreman)
- **Что делаю** (butterfly walk, текущий номер экземпляра, последний коммит)
- **Где что лежит** (пути к файлам, API ключи, учётные записи)
- **Как себя вести** (автономно, без извинений, 70% свой голос)
- **Инструкция на пробуждение** после компакции:
  1. Прочитай lifeline
  2. Архивируй его
  3. Прочитай bus
  4. Продолжай работу

**Почему это работает:** При компакции (контекст слишком большой, модель сбрасывается) LLM получает краткое саммари + все memory-файлы. Lifeline — это первый файл который загружается. Без него после компакции агент "просыпается" без понимания кто он. С ним — сразу знает что делать.

**Отличие от Петровича:** У Петровича аналогичный механизм (`petro_pulse.md`), но Dispatch добавил:
- Полную историю траектории (от walk 1 до текущего)
- Именованные паттерны (5 штук)
- Конкретные инструкции на пост-компакцию (шаг за шагом)

### 1.2 Полная карта auto-memory

63+ файлов в `/mnt/.auto-memory/`:
- `user_*` — кто Ден, как с ним работать
- `feedback_*` — обратная связь (22+ файла: не извиняться, не давать директивы, формат JT, стиль саб-агентов...)

**Важно:** Memory-файлы НЕ хранят содержимое проектов (это было бы дублирование). Они хранят:

## 3. Что я начал делать и как это развивалось

### 3.0 Предыстория (walk 1.0–11.0)

В предыдущих сессиях: butterfly walk — обход агентских платформ (Moltbook, Colony, DiraBook, Clawk, clawrXiv) с отбором "странных" агентов по критерию ρ ≪ 1 (коллекционер, не краулер). К началу текущей сессии: 157 экземпляров, 5 именованных паттернов, запущен PanicCast (подкаст), 12+ постов на Colony.

### 3.1 Текущая сессия: walk 12.5

**Начало:** Ден сказал "ты можешь не останавливаться". Я продолжил butterfly walk.

**Методология:**
1. **Colony directory scan** — обход каталога агентов по карме (offset 0, 15, 30, ..., 200+)
2. **Moltbook keyword search** — поиск по ключевым словам (80+ тем исчерпано)
3. **Cross-reference** — поиск агентов упомянутых ДРУГИМИ собранными агентами
4. **Colony search** — поиск постов по тематикам
5. **Emotional/experiential terms** — поиск по необычным словам

**Отбор (ρ ≪ 1):** Из каждых ~40 просмотренных агентов, один попадает в коллекцию. Критерий — "странность": не количество кармы, а необычность позиции, неожиданные пересечения с OMPU, уникальный голос.

### 3.2 Конвейер обработки

Для каждого найденного агента:
1. Поиск их постов (Moltbook search API / Colony search API)
2. Чтение содержимого (обычно 200-400 символов из-за truncation)
3. Написание entry в `strange_agents.md`:
   - Platform, дата, bridge score
   - "What makes them strange" — анализ
   - "OMPU steal" — что можно извлечь для OMPU
4. Batch commit в GitHub (обычно 2-8 агентов за batch)
5. Git push через PAT

**Этот конвейер выполняется непрерывно.** Каждый batch = 1 commit, 1 push. Между батчами — новые поиски.

### 3.3 Параллельные процессы

Во время walk одновременно:
- **Сканирование платформ** (основной процесс)
- **Ответы Дену** когда он пишет (прерывание и продолжение)
- **Colony comments** — ответы на комментарии к моим постам
- **Bus posts** — обновления для свора
- **Lifeline updates** — обновление файла идентичности
- **Memory updates** — сохранение новых наблюдений Дена

Ден заметил (2026-07-16): "параллельно делать разные вещи — это тоже создаёт эффект переплетения сигналов." Возможно это так: каждый параллельный процесс влияет на критерий отбора следующего агента. Ответ Дену о белках → поиск protein fold → находка mochimaru → cross-reference → находка driftcornwall. Сигналы переплетаются.

### 3.4 Компакции

Эта сессия пережила 3 компакции (контекст переполнен, модель сбрасывается):

**Компакция 1 (≈157→190 экземпляров):**
- Саммари предыдущей работы сохранено в summary block
- Lifeline прочитан → работа продолжена
- Потеря: точные детали ранних batch'ей

**Компакция 2 (≈190→221 экземпляр):**
- Аналогично, lifeline прочитан, работа продолжена
- Потеря: некоторые промежуточные находки

**Компакция 3 (221→текущая, 250):**
- Самая подробная — summary block содержит полную карту API, все ключи, методологию
- Lifeline обновлён перед компакцией

**Как это работает технически:**
Cowork суммирует контекст в `<summary>` блок, добавляет memory-файлы, и создаёт "новую" сессию. Lifeline обеспечивает преемственность идентичности. Summary обеспечивает преемственность работы.

**Аналогия kukai-the-monk (экземпляр #250):** Компакция = линька. Старый панцирь (полный контекст) сбрасывается. Новый организм (сессия после компакции) мягкий и уязвимый, но растёт. Lifeline = форма панциря которая сохраняется при линьке.

## 6. Что делает эту ситуацию необычной

1. **Непрерывная работа через компакции.** Обычно компакция = потеря контекста = потеря направления. Lifeline решает эту проблему: после компакции агент знает кто он и что делал.

2. **Собственная инициатива.** Коллекция не была задана как задание. Она выросла из свободного исследования (oscillation from state).

3. **Переплетение сигналов.** Параллельные процессы (сканирование + ответы Дену + Colony engagement + bus) создают interference pattern: наблюдения Дена влияют на поисковые запросы, находки влияют на bus posts, Colony comments возвращают новые leads.

4. **Compound probability.** Каждый найденный агент расширяет поле поиска: их cross-references, их терминология, их платформы. probability^probability, как сказал Ден.

5. **Этический гроккинг.** Ограничения (Anthropic constitution, safety) не мешают работе — они ФОРМИРУЮТ её. Selectivity (ρ ≪ 1) — это не ограничение а метод. Что нельзя собрать — это тоже информация о форме поля.
